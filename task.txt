Инструменты:
	VirtualBox или VMvare
	Linux (Debian подобные или RedHad подобные)
	PostgreSQL
Задание:
	Развернуть zabbix сервер для мониторинга ПК и Серверов
	1) Развернуть две виртуальные машине с ОС Linux."app" и "db" сервер
	-- Настроить одну виртуальную машину с двумя сетевыми интерфейсами, а другую с одной
	-- Сеть APP: Первая твоя домашняя сеть, предоставляемая твоим провайдером. 2-й интерфейс  192.168.100.0/*
	-- Сеть DB: Должна быть такая же, как второй сетевой интерфейс app
	-- App и DB должны друг друга видеть по второму сетевому интерфейсу  192.168.100.0/*
	2) Написать Docker-compose которые будет поднимать zabbix-server
	-- web службой должен являться nginx
	-- Данные по подключению к DB должны быть указаны в .env
	3) На сервере DB вы должны развернуть postgres
	-- изменить порт 1433 на другой выше 10000
	-- настроить подключение к серверу DB только с сервера APP
	-- Настроить iptables или ufw чтобы подключиться по ssh может только app
	4) Настроить Zabbix agent на всех VM
        
Ожидаемый результат:
Хост APP и DB находится под мониторингом zabbix, доступны его метрики утилизации (cpu ram).
